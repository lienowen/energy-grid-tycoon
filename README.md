# Energy Grid Tycoon

上帝视角城市建设经营游戏。玩家直接在城市地图上选择项目、挑选用地、放置设施，并观察街区亮灯、人口、环境和城市资金随决定发生变化。

## 核心玩家循环

```text
观察城市
→ 选择建设项目
→ 在地图上挑选合适地块
→ 设施直接落地并接入城市
→ 观察居民与城市变化
→ 扩建、关闭或调整发展方向
```

主画面必须始终是城市，而不是数据后台。数据和设置只作为辅助工具，不取代地图操作。

## 当前可玩内容

- 三座按配置解锁的城市地图与最好成绩
- 每座城市拥有不同的社区、产业、沿海、城郊和公共设施用地
- 玩家先选择建设项目，再点击地图上的可用地块完成建设
- 设施位置、运行状态、等级和储电量全部进入存档
- 点击地图中的已建设施，可直接查看、关闭或扩建
- 社区太阳能、城市风电、燃气应急电站、海上风电、先进核能与两级备用电站
- 六项城市发展项目与高级设施解锁
- 四种城市发展方向，可优先照顾居民、产业或环境
- 昼夜用电、天气、收入、人口、居民满意和环境压力模拟
- 备用电自动保存和释放，城市供电线路实时亮起
- 固定种子的城市事件与组合规则
- 下一步提示、城市情况、版本化存档、自动保存、重试和下一城流程
- 响应式桌面与移动端界面
- 全局素材目录、按城市加载和失败降级

## 内容生产标准

新城市原则上只增加：

```text
城市地图配置
+ 地块与初始设施位置
+ 素材引用
+ 目标和规则组合
+ 少量独特行为
```

设施允许建设的区域来自设施配置，地块坐标、区域、用途和初始位置来自城市配置。业务代码不按城市编号分支。

## 开发

```bash
npm install
npm run dev
```

提交前运行完整质量门：

```bash
npm run quality
```

也可以单独运行：

```bash
npm run check:content  # 城市地块、配置、素材引用、重复文件、尺寸和体积预算
npm test               # 地图落位、存档、确定性逻辑与目录校验测试
npm run build          # 严格类型检查和生产构建
```

CI 会按相同顺序执行内容检查、单元测试和生产构建。详细标准见 `docs/QUALITY_GATE.md`。

## 架构

- `core`：应用、会话、状态、地图配置、版本化存档、领域事件与城市进度
- `systems`：地图落位、电力、储能、经济、事件、目标、城市发展、方向、引导和统计
- `rules`：Condition、Action、Modifier 与 Behavior 的组合协议和注册表
- `gameplay`：少量需要代码实现的独特行为
- `buildings`：设施模型、位置、工厂、集合和运行时快照
- `data`：城市地图、地块、设施、事件、发展项目和方向参数
- `resources`：全局素材目录、元数据、预加载和城市素材规划
- `ui`：上帝视角城市世界、地图输入与辅助信息

完整边界与扩展规则见 `docs/ARCHITECTURE.md`。

核心原则：

- 世界先于面板
- 玩家行动先于数据展示
- 稳定能力 = 引擎
- 重复变化 = 配置
- 新城市 = 配置 + 素材 + 规则组合 + 少量独特行为
- 素材 = 全局资源 ID，不在业务代码中写路径
- 特殊玩法 = 注册行为，不复制城市方法
- 存档 = 状态与运行时快照，不复制配置
- 每个新规则和行为 = 确定性测试 + 内容校验
