# 全息城市沙盘实现标准

主视图不再由 DOM 卡片拼成城市地图。城市世界由持续存在的 Canvas 场景渲染，DOM 只负责状态栏、目标、提示、建造栏和二级工具。

## 数据流

```text
GameManager / Systems / Rules
            ↓
       GameViewModel
            ↓
      CitySceneMapper
            ↓
       CitySceneState
            ↓
  HologramCanvasRenderer
```

- `GameManager` 不知道镜头、道路、车辆、光效、动画和 Canvas。
- `CitySceneMapper` 是纯数据映射，不操作 DOM。
- `CitySceneVisuals` 根据关卡 ID、主题和地块配置，确定性生成道路、街区模型和车流种子。
- `HologramSandbox` 管理镜头、鼠标、触控、缩放和命中检测。
- `HologramCanvasRenderer` 只根据只读场景状态绘制。
- HUD 通过 `MayorDashboard` 更新，但不会销毁沙盘 Canvas。

## 玩家操作闭环

```text
底部选择设施
→ CitySceneMapper 计算合法地块
→ 沙盘高亮可建位置
→ 玩家点击地块
→ GameManager.build(configId, plotId)
→ 存档记录 placementId
→ 沙盘播放地基、脚手架、主体升起和线路接通动画
```

## 场景分层

渲染顺序固定为：

1. 指挥室背景与透视网格
2. 全息沙盘底座
3. 城市分区地台
4. 道路网络
5. 居民楼、产业楼、公共设施和公园
6. 输电线路与流动光点
7. 城市核心
8. 建设地块与能源设施模型
9. 道路车辆
10. 分区停电警示、扫描光和状态反馈
11. 角落场景信息

逻辑系统只产生状态。停电、空气恶化、储能工作、建设完成等视觉变化由场景层表达。

## 城市生活表现

- 道路和街区模型由稳定种子生成，同一关卡、同一配置每次显示一致。
- 车流密度由人口、城市亮灯率和道路等级决定。
- 居民楼窗灯受昼夜、分区供电率和稳定种子控制。
- 产业区拥有烟囱、暖色工作灯和更宽道路。
- 公园、树木与低层设施随城市主题变化。
- 供电不足时，分区按照城市主题的民生优先级逐步变暗。
- 严重停电区域显示脉冲边界，不把警告藏进数据面板。

## 设施模型标准

当前设施不再直接把资源图标贴在地图上，而是通过 `FacilityModelRenderer` 绘制独立等距模型：

- 社区光伏：阵列式倾斜面板
- 城市风机：塔筒和持续旋转叶片
- 燃气应急站：厂房、烟囱和工作烟气
- 城市备用电站：储能柜与实时电量条
- 大型储电中心：多排储能柜园区
- 海上风电区：水面平台与双风机
- 先进核能电站：冷却塔、反应堆穹顶和运行脉冲

未知的新设施仍使用通用等距模型，避免配置内容导致场景崩溃。新增正式设施应在模型注册表中补充独立模型，并添加确定性测试。

## 配置扩展

`LevelWorldPresentationConfig` 提供可选的 `sandbox`：

```ts
sandbox: {
  startZoom: 1,
  minZoom: 0.72,
  maxZoom: 1.75,
  startOffsetX: 0,
  startOffsetY: 18
}
```

地块继续来自关卡配置，可增加：

```ts
{
  id: 'north-community',
  x: 24,
  y: 28,
  elevation: 1.2,
  footprint: { width: 12, height: 9 },
  zone: 'neighborhood',
  accepts: ['generation', 'storage']
}
```

道路、街区和车流的默认布局从地块配置确定性生成。未来需要手工美术布局时，可以新增可选道路与装饰配置，但渲染代码不得根据 `level.id` 分支。

## 性能要求

- Canvas 不随模拟 tick 重建。
- 设备像素比最高使用 2，避免超高分屏浪费。
- 场景状态更新与动画帧分离。
- 道路、街区和车辆不进入游戏存档，只由关卡配置与游戏状态重建。
- 命中区域由渲染器统一返回，输入层不复制地图算法。
- 初始设施不播放施工动画，新增设施才播放。
- 视觉随机必须使用稳定种子，禁止使用 `Math.random()`。
