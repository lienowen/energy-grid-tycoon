# Energy Grid Tycoon

城市能源模拟经营游戏。玩家建设发电与储能设施，升级资产，研发能源科技，制定城市政策，并在需求高峰和随机事件中维持城市发展。

## 当前可玩内容

- 三座按配置解锁的城市关卡与最佳评分记录
- 光伏、风电、燃气、海上风电、先进核能与两级储能建筑
- 建筑独立等级、升级成本、产能与维护成长
- 六项数据驱动科技树与高级建筑解锁
- 四项城市政策，可改变需求、收益、满意度、污染、维护和研发效率
- 昼夜负荷、发电波动、电网损耗、经济、人口和污染模拟
- 储能自动充放电、效率损耗、容量与功率升级
- 固定种子的动态城市事件与规则组件
- 最近 48 个调度周期的供需趋势和累计运营统计
- 版本化本地存档、旧存档迁移、自动存档、读档、重试和下一关流程
- 胜利、失败、战役选择与响应式桌面/移动端 UI
- 全局素材目录、启动预加载、关卡按需预加载和失败降级

## 开发

```bash
npm install
npm run dev
```

提交前运行完整质量门：

```bash
npm run quality
```

也可以单独运行：

```bash
npm run check:content  # 配置、素材引用、重复文件、尺寸和体积预算
npm test               # 确定性逻辑与目录校验测试
npm run build          # 严格类型检查和生产构建
```

CI 会按相同顺序执行内容检查、单元测试和生产构建。详细标准见 `docs/QUALITY_GATE.md`。

## 架构

- `core`：应用、会话、状态、版本化存档、领域事件与战役进度
- `systems`：电力、储能、经济、事件、目标、科技、政策、升级、遥测等公共算法
- `rules`：Condition、Action、Modifier 与 Behavior 的组合协议和注册表
- `gameplay`：少量需要代码实现的独特行为
- `buildings`：建筑模型、工厂、集合和运行时快照
- `data`：关卡、建筑、事件、科技和政策参数
- `resources`：全局素材目录、元数据、预加载和关卡素材规划
- `ui`：界面渲染、输入与表现反馈

完整边界与扩展规则见 `docs/ARCHITECTURE.md`。

核心原则：

- 稳定能力 = 引擎
- 重复变化 = 配置
- 新关卡 = 配置 + 素材 + 规则组合 + 少量独特行为
- 素材 = 全局资源 ID，不在业务代码中写路径
- 特殊玩法 = 注册行为，不复制关卡方法
- 存档 = 状态与运行时快照，不复制配置
- 每个新规则和行为 = 确定性测试 + 内容校验
