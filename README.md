# Energy Grid Tycoon

城市市长模拟经营游戏。玩家接手一座城市，听取居民需要，安排供电建设，在市政资金、居民生活、城市发展和空气质量之间做决定。

## 当前可玩内容

- 三座按配置解锁的城市与最好市民评价
- 以城市全景为主的游戏画面，建筑直接出现在地图中
- 市政秘书根据当前局面告诉玩家下一步可以做什么
- 社区太阳能、城市风电、燃气应急电站、海上风电、先进核能与两级备用电站
- 设施独立扩建、开关和日常开支管理
- 六项城市升级与高级项目解锁
- 四种施政方向，可优先照顾居民、工业或环境
- 昼夜用电、天气、收入、人口、居民满意和空气压力模拟
- 备用电力自动储存和释放
- 固定种子的城市突发事件与组合规则
- 城市报告、版本化存档、自动保存、重试和下一城流程
- 响应式桌面与移动端界面
- 全局素材目录、按城市加载和失败降级

## 开发

```bash
npm install
npm run dev
```

提交前运行完整质量门：

```bash
npm run quality
```

也可以单独运行：

```bash
npm run check:content  # 配置、素材引用、重复文件、尺寸和体积预算
npm test               # 确定性逻辑与目录校验测试
npm run build          # 严格类型检查和生产构建
```

CI 会按相同顺序执行内容检查、单元测试和生产构建。详细标准见 `docs/QUALITY_GATE.md`。

## 架构

- `core`：应用、会话、状态、版本化存档、领域事件与城市进度
- `systems`：电力、储能、经济、事件、目标、城市升级、施政方向、引导和统计算法
- `rules`：Condition、Action、Modifier 与 Behavior 的组合协议和注册表
- `gameplay`：少量需要代码实现的独特行为
- `buildings`：设施模型、工厂、集合和运行时快照
- `data`：城市、设施、事件、升级和施政参数
- `resources`：全局素材目录、元数据、预加载和城市素材规划
- `ui`：市长视角的城市世界、交互面板与表现反馈

完整边界与扩展规则见 `docs/ARCHITECTURE.md`。

核心原则：

- 稳定能力 = 引擎
- 重复变化 = 配置
- 新城市 = 配置 + 素材 + 规则组合 + 少量独特行为
- 素材 = 全局资源 ID，不在业务代码中写路径
- 特殊玩法 = 注册行为，不复制城市方法
- 存档 = 状态与运行时快照，不复制配置
- 每个新规则和行为 = 确定性测试 + 内容校验
