# Energy Grid Tycoon

上帝视角城市建设经营游戏。玩家站在未来城市指挥台前，直接拖动和缩放全息城市沙盘，选择建设项目、挑选用地、放置设施，并观察街区亮灯、人口、环境和城市资金随决定发生变化。

## 核心玩家循环

```text
观察全息城市
→ 选择建设项目
→ 沙盘高亮可用地块
→ 点击地块让设施升起
→ 供电线路接通并观察城市变化
→ 扩建、关闭或调整发展方向
```

主画面始终是可操作的城市沙盘，而不是数据后台。数据和设置只作为辅助工具，不取代世界操作。

## 当前可玩内容

- 持续存在的 Canvas 全息城市沙盘，不会随模拟 tick 重建
- 鼠标拖动、滚轮缩放、双击归位和移动端双指缩放
- 3/4 俯视投影、悬浮底座、立体分区、电流光点和扫描效果
- 建设施工升起动画、风机旋转、燃气烟气和储能脉冲反馈
- 供电不足时分区灯光变暗，环境恶化时城市出现暖色雾霾
- 三座按配置解锁的城市地图与最好成绩
- 每座城市拥有不同的社区、产业、沿海、城郊和公共设施用地
- 玩家先选择建设项目，再点击沙盘上的可用地块完成建设
- 设施位置、运行状态、等级和储电量全部进入存档
- 点击沙盘中的已建设施，可直接查看、关闭或扩建
- 社区太阳能、城市风电、燃气应急电站、海上风电、先进核能与两级备用电站
- 六项城市发展项目与高级设施解锁
- 四种城市发展方向，可优先照顾居民、产业或环境
- 昼夜用电、天气、收入、人口、居民满意和环境压力模拟
- 备用电自动保存和释放，城市供电线路实时亮起
- 固定种子的城市事件与组合规则
- 下一步提示、城市情况、版本化存档、自动保存、重试和下一城流程
- 响应式桌面与移动端 HUD
- 全局素材目录、按城市加载和失败降级

## 内容生产标准

新城市原则上只增加：

```text
城市地图配置
+ 沙盘镜头与地块高度配置
+ 素材引用
+ 目标和规则组合
+ 少量独特行为
```

设施允许建设的区域来自设施配置；地块坐标、区域、用途、高度、初始位置和镜头参数来自城市配置。业务代码不按城市编号分支。

## 开发

```bash
npm install
npm run dev
```

提交前运行完整质量门：

```bash
npm run quality
```

也可以单独运行：

```bash
npm run check:content  # 城市地块、配置、素材引用、重复文件、尺寸和体积预算
npm test               # 沙盘投影、地图落位、存档、确定性逻辑与目录校验测试
npm run build          # 严格类型检查和生产构建
```

CI 会按相同顺序执行内容检查、单元测试和生产构建。详细标准见 `docs/QUALITY_GATE.md`。

## 架构

- `core`：应用、会话、状态、地图配置、版本化存档、领域事件与城市进度
- `systems`：地图落位、电力、储能、经济、事件、目标、城市发展、方向、引导和统计
- `rules`：Condition、Action、Modifier 与 Behavior 的组合协议和注册表
- `gameplay`：少量需要代码实现的独特行为
- `buildings`：设施模型、位置、工厂、集合和运行时快照
- `data`：城市地图、地块、设施、事件、发展项目和方向参数
- `resources`：全局素材目录、元数据、预加载和城市素材规划
- `presentation`：把游戏状态映射成只读沙盘场景状态
- `ui/world`：Canvas 镜头、输入、分层渲染和命中检测
- `ui`：顶部 HUD、任务、秘书提示、建造栏和二级工具

全息沙盘边界见 `docs/HOLOGRAM_SANDBOX.md`，完整架构规则见 `docs/ARCHITECTURE.md`。

核心原则：

- 世界先于面板
- 玩家行动先于数据展示
- Canvas 负责城市世界，DOM 只负责 HUD
- 稳定能力 = 引擎
- 重复变化 = 配置
- 新城市 = 配置 + 素材 + 规则组合 + 少量独特行为
- 素材 = 全局资源 ID，不在业务代码中写路径
- 特殊玩法 = 注册行为，不复制城市方法
- 存档 = 状态与运行时快照，不复制配置
- 每个新规则和行为 = 确定性测试 + 内容校验
