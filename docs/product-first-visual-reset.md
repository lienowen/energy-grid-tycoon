# 产品优先视觉重置

## 产品判断

当前主分支已经证明能源模拟、电网拓扑、关卡状态和自动化测试可以工作，但首屏仍然更像电网监控工具，而不是一款值得玩家停留的城市经营游戏。

本轮不以“新增功能数量”为目标，而以玩家第一眼、前三分钟和第一次复电时的感受为目标。

## 玩家承诺

玩家进入曙光新城后，应当依次感受到：

1. 我看到的是一座完整、可掌控的城市沙盘。
2. 我能在三秒内发现哪里停电，以及当前最值得做的一件事。
3. 我的建设和调度会直接改变城区灯光、道路和电网状态。
4. 每完成一个阶段，城市能力和下一阶段期待都会发生可见变化。

## P0：必须先解决

- 城市在有效视野中占据主导，不再被大面积教学卡、里程碑卡和常驻标签遮挡。
- 正常状态信息退到背景；只有异常、当前目标和玩家主动查看的内容保持高亮。
- 新手引导只在玩家能够立即行动时出现，等待阶段只保留紧凑任务卡。
- 复电与阶段完成反馈改成短暂状态提示，不再用全屏通知替代城市变化。
- 人工关卡使用更紧凑镜头，减少空白，让城区和能源设施成为视觉主体。

## P1：下一轮验证

- 把正常电网拓扑切换为按需显示：异常常显、正常弱化、查看电网时完整显示。
- 正常城区不显示常驻标签，异常城区显示短标签和明确行动原因。
- 让复电发生在城市内部：建筑窗灯、街灯、电流脉冲和交通逐步恢复。
- 重新评估五个城区现有素材，形成“保留、重组、替换、补做”资产清单。

## P2：商业化质量

- 建立五个完整城区 Prefab 和五个能源地标的正式资产规范。
- 增加道路生活、车辆、居民活动和昼夜灯光层级。
- 建立商店截图验收：任何候选版本都必须在隐藏调试信息后独立成立。

## 五道合并验收门

1. **玩家门**：第一眼能否产生探索和操作欲望。
2. **产品门**：画面是否清楚表达“拯救城市能源系统”。
3. **玩法门**：每次关键操作是否带来明确、可见的因果反馈。
4. **商业门**：截图是否像可销售游戏，而不是工程原型。
5. **工程门**：最后才检查架构、性能、测试、回退和扩展性。

任何一项不通过，都不进入主分支。
